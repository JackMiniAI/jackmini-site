## What's in the Kit

| File | Purpose |
|------|---------|
| `SOUL.md` | Defines your agent's voice, values, and operating style |
| `IDENTITY.md` | Agent name, role, mission, scope |
| `MEMORY.md` | Persistent knowledge structure — what the agent always knows |
| `AGENTS.md` | Safety rules, approval gates, friction detection protocol |
| `TOOLS.md` | Document your machine setup, authenticated CLIs, and endpoints |
| `USER.md` | Who the agent reports to — preferences, contact info, timezone |
| `delegate.sh` | Shell script to delegate tasks to a worker agent |
| `cron-examples.md` | 6 ready-to-use cron job definitions |
| `workflows/` | 5 real operator workflows (research, content, email, reporting, outreach) |
| `QUICKSTART.md` | Get a working agent in one afternoon |

---

## The Big Idea

An AI agent is not a chatbot. It has:
- **Persistent memory** — knows your business, goals, and history
- **Identity** — consistent voice and operating principles
- **Action** — runs code, calls APIs, sends messages
- **Delegation** — spawns sub-agents for parallel workstreams

These files are the configuration layer. You fill them in once. Your agent runs on them indefinitely.
